异常处理

1.final，finally,finalize 的区别

final 是最终的，修饰类，方法，属性都是最终的，修饰类不可被继承，修饰方法不可被重写，修饰属性不可被修改

finally，最终会执行，配合 try catch 执行

finalize，在 objec 类中，释放内存资源的

2.Error 和 Exeption 有什么区别？

Error 和 Exception 都是 Throwable 的子类，区别就是 Error 比 Exception 更严重，不易通过后续代码处理和修复，不应该被代码 catch 到的错误。而 Exception 是指严重程度较轻的程序问题，相较于 Error 更容易被后续代码处理和修复的，有些需要通过 try catch 代码处理，有些不建议通过 try catch 代码处理。

3. 运行时异常与受检异常有什么区别？

运行时异常与受检异常是同一级别的

异常类和任何不是 RuntimeException 的子类的子类都是检查异常，检查的异常需要在方法和构造函数的 thtows 子句中声明.
